CD取り込み自動化ワークフロー・マスターGUI 要件定義書 (v8)

本書の位置づけ: 本ドキュメントは、ユーザー（定義者本人）のCD取り込みワークフローを管理するGUIアプリケーションの決定版要件定義書である。v8では、実装段階で具体化したUI要件（サイドバー、リスト表示、強調表示など）と、最新のデータスキーマを完全に反映した。
最終更新: 2025年11月9日

1. プロジェクト概要

1.1. 目的とゴール

目的: 個人の趣味におけるCDリッピング作業全10ステップを、GUIナビゲーションにより標準化・効率化する。

ゴール: 迷いなく、手戻りなく、確実に作業を完遂できる状態を作る。

1.3. 開発・動作環境

  • OS: Windows 10 / 11 (64bit)
  • 言語: Python 3.12 (PySide6)
  • 必須ライブラリ: PySide6, mutagen, send2trash

2. システム構成

2.1. 設定・状態管理

  • config.ini: 実行ファイルと同階層。ツールパス自動検出（PATH利用）時は警告を表示。
  • state.json: ステップ完了時、重要操作時、アプリ終了時に保存。

2.2. ディレクトリ構造要件

work/
  └─ [Artist] - [Album]/          <-- アルバムフォルダ
       ├─ state.json              <-- 進捗管理ファイル
       ├─ 01 - Title.flac         <-- ★完成品FLAC
       ├─ 02 - Title (Inst).flac  <-- ★Demucs成果物 (Inst付与)
       │
       ├─ _flac_src/              <-- (Step1一時) 生FLAC置き場
       ├─ htdemucs_ft/            <-- (Step2一時) Demucs出力
       ├─ _aac_output/            <-- (Step4成果) AAC出力先
       ├─ _opus_output/           <-- (Step5成果) Opus出力先
       ├─ _artwork_resized/       <-- (Step6成果) リサイズ画像出力先
       ├─ _final_flac/            <-- (Step8成果) 最終アーカイブ用
       └─ _logs/                  <-- 自動処理ログ

3. 機能要件詳細

実装状況凡例: ✅ 完全実装済み | 🔶 部分実装済み | ❌ 未実装

3.1. マスターGUI 基本機能

  • アルバム管理 (✅): work 監視、リスト表示、進捗表示。
  • 禁止文字処理 (✅): ファイル名生成時に \/:*?"<>| を全角等に自動置換。

3.2. ワークフロー・ステップ詳細仕様

Step 0: Music Center 取り込みガイド (✅)

  • Music Center 自動起動、手順案内。

Step 1: 新規取り込み (✅)

  • UI強化: 非ネイティブダイアログに「Music Center」「Desktop」等のサイドバーショートカットを追加。
  • 自動検出: 選択フォルダ配下をスキャンし、.flac が存在するフォルダをアルバムとして認識。
  • 安全性: 二段階コピー (shutil.copytree → send2trash)。

Step 2: Demucs処理 (✅)

  • 高度な自動除外: キーワード＋ペア曲推定。
  • リネーム: 成果物に (Inst) を付与。

Step 3: FLAC完成 (✅)

  • UI要件:
    • 左（元ファイル）・右（現ファイル）の並列リスト形式で表示。
    • ドラッグ＆ドロップ等で手動紐づけ変更が可能（推奨）。
    • Inst曲強調: Instrumentalと判定された曲には、視覚的に目立つバッジ（例：金色）を付与する。
  • 自動処理: 紐づけ、アートワーク検査(mutagen)、ReplayGain測定(foobar2000)。

Step 4 & 5: 変換 (AAC / Opus) (✅)

  • 厳密なバリデーション: 出力ファイル数が期待値と不一致なら警告ダイアログを表示。

Step 6: アートワーク縮小 (🔶)

  • 未実装(❌): 自動実行、アートワーク無し時の自動スキップ。

Step 7: 最終転送 (✅)

  • 3サブステップ (FLAC, AAC, Opus) のタブUI。

Step 8, 9, 10: 最終処理 (🔶)

  • 未実装(❌): 完了後のクリーンアップ（ゴミ箱移動）提案。

4. データ定義詳細

4.1. `state.json` スキーマ定義 (完全版)

{
  "version": "1.0",
  "hasArtwork": true,             // (Step3自動判定)
  "currentStep": 4,
  "tracks": [
    {
      "id": "track_01",
      "originalFile": "01.flac",
      "finalFile": "01 - Title.flac",
      "instrumentalFile": "02 - Title (Inst).flac", // Instファイル名
      "hasInstrumental": true,    // Instペアが存在するか
      "isInstrumental": false,    // これ自体がInstか
      "demucsTarget": true        // Demucs処理対象だったか
    }
  ],
  "paths": {
    "rawFlacSrc": "_flac_src",
    "demucsOutput": "C:/.../htdemucs/Album",
    "aacOutput": "_aac_output",
    "opusOutput": "_opus_output",
    "artworkResized": "_artwork_resized",
    "finalFlac": "_final_flac"
  }
}
